security-system — Ubiquitous Language Glossary

Single source of truth for ubiquitous-language terms. Each term has a stable kebab-slug used as the anchor id. Tests link here via @Concept("<slug>") (Allure), and Javadoc links here via the matching anchor.

Registration

Creating a new user from an email and a password, subject to email and password validation.

Credential verification

Checking presented credentials against a stored user: the user must exist and the password hash must match. Any other outcome is a failure.

Brute-force guard

Protection that rejects authentication attempts from an IP address that has an active block, or that has reached the failure limit within the configured window (then a new block is created). Backed by failed-authentication and authentication-block records.

Session tokens

The pair of tokens (refresh + access, with expirations) that represents an active session, minted for an email and persisted.

Refresh-token rotation

Renewing a session by consuming the presented refresh token (single use): the existing session is deleted and, if the token had not expired, a fresh set of session tokens is issued.
